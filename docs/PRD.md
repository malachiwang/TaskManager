The app should automatically handle:- last-done detection- days-since calculation- urgency scoring- historical continuity- monthly/archive storage- dashboard summaries- tendency/behavior analyticsThe system must minimize active engagement. It should track first and foremost.CURRENT SYSTEM CONTEXTThe user currently uses a long-running Google Sheets productivity/todo tracker.The sheet includes:- task rows grouped by sections such as:  - Habits  - Career  - Projects  - Music  - Miscellaneous  - Health Reminders  - Portal Network / Key sections- metadata columns including:  - Priority  - Status  - Category  - Task  - Subtask  - Freq  - Days S  - Urg  - Manual  - Notes- a wide dated checkbox grid across days/months- visible date headers such as May/June day numbers- recurring checkboxes per task/day- manual date overrides for tasks whose last completion predates the current working sheet- urgency values derived from days since, frequency, and priority- notes such as “TEMPORARY HIATUS,” “Currently no need to do,” etc.- a priority/status key with categories like:  - Background  - Passive  - Focus  - VitalThe sheet has worked for years because it is static, organic, and low-friction. The problem is that monthly recycling/resetting breaks automatic last-done detection, and formulas/conditional formatting become brittle as the tracker grows.CORE USER GOALSThe app should help the user:1. Track everything important without forgetting.2. Maintain habits and recurring responsibilities.3. Prevent neglect across life categories.4. Reduce cognitive load by surfacing what matters.5. Maximize productive output without requiring daily planning.6. Preserve a spreadsheet-like vibe rather than forcing a rigid productivity-app workflow.7. model.Core idea:- priority determines the floor- days_since / interval determines growth- urgency approaches 10- urgency gets very close to 10 around double the ideal interval- high-priority tasks remain noticeably urgent even after completion- formula should avoid exploding infinitelyDefinitions:P = priorityD = days since last completionI = interval/frequency in daysk = growth constant, default around 2Max urgency = 10Priority base mapping:If P >= 8:  base = 0.8 + 0.05 * (P - 8)If P >= 5:  base = 0.35 + 0.15 * (P - 5)Else:  base = 0.05 + 0.1 * (P - 1)Floor:floor = base / 2Growth:growth = 1 - exp(-k * (D / I))Urgency:urgency = 10 * (floor + (1 - floor) * growth)Default k:k = 2Why k = 2:At D/I = 2:growth = 1 - exp(-4) ≈ 0.9817So urgency is about 98% of the way from floor to max at double the ideal interval.Important:- “TODAY” is display text only- internally D = 0 for today- formulas should never rely on text values for mathURGENCY INTERPRETATIONSuggested bands:- 0–3: low / fresh- 3–6: noticeable- 6–8: high- 8–9.5: near-max- 9.5–10: critical / extremely highBecause urgency asymptotically approaches 10, it may rarely equal exactly 10 unless
